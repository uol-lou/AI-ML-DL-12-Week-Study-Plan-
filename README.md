## ✅ 学习建议与常见问题

### 学习原则

1. **70%实践 + 30%理论**：先会用，再深究原理
2. **小步快跑**：每周完成小项目，积累成就感
3. **遇到困难先搜索**：90%的问题别人都遇到过
4. **记录学习笔记**：每周写总结，加深理解
5. **参与社区**：GitHub、Kaggle、技术论坛

### 时间分配建议

- **平日（周一至周五）**：每天1-1.5小时
  - 观看视频/阅读文档：30-45分钟
  - 动手编码实践：30-45分钟
- **周末**：每天3-4小时
  - 完成本周实践项目
  - 整理代码和笔记
  - 探索额外资源

### 常见困难及应对

**Q1：数学基础不够怎么办？**

- 边学边补，重点关注：线性代数（矩阵运算）、微积分（求导）、概率统计
- 推荐：3Blue1Brown的线性代数和微积分系列视频

**Q2：代码报错不知道怎么解决？**

- 仔细阅读错误信息
- 复制错误信息到Google/Stack Overflow搜索
- 检查数据维度、类型是否匹配
- 使用print()调试中间结果

**Q3：模型效果不好怎么办？**

- 先检查数据质量和预处理
- 尝试简单模型建立baseline
- 逐步增加复杂度
- 学习特征工程和超参数调优

**Q4：学习进度跟不上怎么办？**

- 不要焦虑，可以延长时间线
- 优先完成核心概念和项目
- 跳过过于理论的数学推导
- 专注于实践应用

**Q5：感觉进步缓慢怎么办？**

- 回顾第1周和现在的对比
- 参加竞赛获得排名反馈
- 与同学交流学习心得
- 设定小目标并庆祝达成

### 进阶方向（12周后）

1. **深入某个领域**：CV、NLP、推荐系统、强化学习
2. **研究前沿技术**：Transformer、Diffusion Models、大语言模型
3. **参加高级竞赛**：Kaggle竞赛、AI算法大赛
4. **阅读经典论文**：从Papers with Code选择方向
5. **贡献开源项目**：参与或创建开源ML/DL项目
